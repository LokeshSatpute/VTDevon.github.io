Use Cases

Event Organizer Use Cases

  • The organizer can create events by specifying details such as date, time, location, ticket types, prices, and capacity. This requires an “organizer” profile login, and once created, the event is available to attendees.
  • The organizer can monitor and manage ticket sales in real time, viewing details like ticket types sold and remaining capacity. This functionality is available post-event creation, with real-time updates shown on the organizer's frontend.
  • The organizer can add merchandise and concessions for attendees to pre-order or purchase at the event. Items listed by the vendor become available for purchase.
  • The organizer has access to an analytics dashboard displaying real-time insights on ticket sales, attendee check-ins, merchandise sales, and vendor performance.

Attendee Use Cases

  • The attendee can search and browse events based on categories, dates, locations, and popularity. Only listed events are visible to attendees; in-progress events are hidden.
  • The attendee can buy tickets and receive a unique QR code after completing a valid payment. The QR code is generated upon successful payment.
  • The attendee can pre-order food, drinks, and merchandise, which are linked to their QR code for easy access at the event. The QR code is updated once the order is accepted.
  • The attendee checks in by scanning their QR code, registering them as present and granting vendors access to pre-ordered items. Additional purchases can be made on-site, updating the QR code accordingly.

Vendor Use Cases

  • Vendors can access a list of orders linked to attendee QR codes, fulfill these orders upon scan, and update order statuses.
  • Vendors can update inventory levels for food, drinks, and merchandise, providing real-time availability for organizers and attendees to view.
